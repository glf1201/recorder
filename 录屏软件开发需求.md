\# Windows录屏软件开发需求说明书



\## 一、项目概述



开发一款基于 C#（.NET 8）+ FFmpeg 的 Windows 桌面录屏软件，用于长期无人值守录制电脑屏幕及系统声音，要求采用 MVVM 架构开发，代码结构清晰，符合企业级项目规范，所有核心功能需具备详细注释和日志记录，便于后续维护和二次开发。



软件主要应用于：



\* 安防值守中心

\* 监控室值班电脑

\* 企业办公电脑留痕

\* 长时间运行环境



要求软件能够长期稳定运行，支持 7×24 小时连续录制，并具备自动切片、自动清理、异常恢复等能力。



\---

\## 支持系统



仅支持：

\# 三、核心功能需求



\## 1. 自动录屏



\### 功能要求



软件启动后自动开始录屏。



支持：



\* 全天候连续录制

\* 无需人工干预

\* 长期稳定运行



\### 异常恢复



发生以下情况时应自动恢复录制：



\* FFmpeg异常退出

\* 录制进程崩溃

\* 系统休眠后恢复

\* 显示器关闭后恢复



恢复时间：



\* 不超过30秒



\---



\## 2. 视频切片



\### 切片规则



采用自然时间对齐方式切片。



每段：



\* 10分钟



例如：



```text

02:00 - 02:10

02:10 - 02:20

02:20 - 02:30

```



\### 首段处理



若软件启动时间不是整10分钟节点：



例如：



```text

启动时间：14:03

```



则：



```text

14:03 - 14:10

```



为第一段。



之后：



```text

14:10 - 14:20

14:20 - 14:30

```



全部保持10分钟整点对齐。



\### 文件命名



格式：



```text

YYYY-MM-DD\_HH-mm.ext

```



示例：



```text

2026-06-06\_14-20.mkv

2026-06-06\_14-30.mkv

```

\## 11. 自动清理录像



\### 保留时间



默认：



```text

7天

```



可配置范围：



```text

3\~30天

```



\### 清理执行时间



默认：



```text

15:00

```



允许用户自定义：



```text

00:00 \~ 23:59

```



任意时间。



\### 清理策略



每天在设定时间执行一次。



删除：



```text

超过保留天数的视频文件

```



\### 启动补偿机制



若软件在计划清理时间未运行。



例如：



```text

计划时间：15:00

启动时间：18:00

```



则软件启动后10分钟内自动补执行一次清理任务。

\# 四、磁盘空间保护



\## 剩余空间预警



当录像目录所在磁盘剩余空间低于：



```text

5GB

```



时：



\* 主界面显示告警

\* 写入日志



\---



\## 自动保护



当磁盘剩余空间低于：



```text

1GB

```



时：



自动删除最旧录像文件。



直到：



```text

剩余空间 ≥ 5GB

```



\---



\## 删除原则



优先删除：



```text

最早生成的录像文件

```



禁止删除：



```text

当前正在录制的文件

```

\# 五、视频完整性保护



\## 异常退出保护



发生以下情况时：



\* 软件崩溃

\* Windows死机

\* 系统断电

\* 强制结束进程



录像文件应尽可能保持可播放状态。



\---



\## 启动自检



软件启动时自动检查：



\* 上次录制是否异常结束

\* 是否存在未正常关闭的视频文件



发现异常文件时：



\* 自动尝试修复

\* 修复失败写入日志

例如：



```text

WatchDog.exe
```

若主程序异常退出：



```text

```



\### 监控对象



\* 主程序崩溃

\* FFmpeg异常退出

\* 主程序无响应
