# 录屏软件项目

## 技术栈
- C#
- .NET 8
- WPF
- FFmpeg

## 当前实现
- `RecorderApp`：主程序，负责配置管理、FFmpeg 录制、10 分钟自然时间切片、异常恢复、磁盘清理和基础界面。
- `WatchDog`：守护程序，基于心跳文件监控主程序存活状态，异常时在 5 秒轮询周期内拉起主程序。
- 启动时自动检查 `FFmpeg`，缺失时尝试自动下载并部署到 `Tools/ffmpeg/bin/`。
- 音频设备支持自动枚举系统播放设备，并在界面中以下拉方式选择。
- 主界面已改为状态中心，只显示录制状态、当前文件、磁盘与环境摘要。
- 设置改为独立弹窗，按“录制参数 / 设备与环境 / 存储与维护 / 系统与安全”分类展示。
- 主界面支持展示最近录像和最新日志，便于值守人员快速确认运行情况。
- 程序支持系统托盘常驻，最小化或关闭窗口后可继续在后台运行。
- 托盘菜单支持直接打开主界面、设置、录像目录、日志目录，以及刷新状态。
- 设置窗口已支持目录选择器和密码输入控件，避免手工填写路径或明文输入密码。
- 设置窗口采用标准“确定 / 取消”交互，取消或直接关闭窗口会回滚未保存修改。
- 托盘图标会根据当前录制状态动态变色，便于值守时快速识别运行状态。

## 目录说明
- `src/RecorderApp`：主录屏程序
- `src/WatchDog`：守护程序
- `Tools/ffmpeg/bin/ffmpeg.exe`：建议放置 FFmpeg 可执行文件

## 运行前提
1. 安装 .NET 8 SDK 和对应 Windows Desktop Runtime。
2. 推荐联网运行首启，让程序自动下载 `FFmpeg`；也可以手动将 `ffmpeg.exe` 放入 `Tools/ffmpeg/bin/`，或加入系统 `PATH`。
3. 在 Windows 10/11 上编译运行。

## 说明
- 默认保存目录为程序目录下 `Record/`。
- 默认格式为 `mkv`，按日期自动分目录，文件名为 `yyyy-MM-dd_HH-mm.ext`。
- 启动后可自动开始录制，也支持设置退出密码。
- 当前版本已接入 `FFmpeg` 自动部署与音频设备枚举；安装包封装与升级覆盖安装仍未接入。
